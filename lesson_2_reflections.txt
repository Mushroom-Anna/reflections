1、初始化版本库时会发生什么情况？为什么你需要这样做？
git log 命令报错，没有历史commit

2、暂存区与工作目录和版本库有何不同？你认为它具有什么价值？
工作目录的文件可以一个个add进staging area, 然后一起放入版本库。
可以让不同文件一次性放入版本库里，一次改变用一个commit

3、如何使用暂存区确保为每项逻辑更改进行一次提交？
git add + filename 可以存入暂存区，git commit 将暂存区所有改动提交

4、在哪些情况下，分支可以帮助你有序地组织历史记录？分支将如何提供帮助？
对某个功能改动的时候，创建不同版本的时候。
项目向两个不同的方向前进的时候使用分支很有帮助

5、图表如何帮助你直观显示分支结构？
可以直观地知道每条分支是做什么的，从哪里开始分支

6、将两个分支合并到一起有何结果？为什么我们在图表中表示它的方式是那样的？
根据原始文件，两个分支的内容会合并
